# DISCORD_SERVER.md

## Overview

This document defines the official structure, guidelines, backlog, and growth strategy for the the **Trophi.gg Discord Server**.  
It is designed as a **hybrid document**: part documentation, part executable backlog to be used by whoever sets up and maintains the server.

---

## 1. Purpose of the Server (Pre-Launch ‚Üí Post-Launch)

The Trophi.gg Discord server exists to:

- Build an early community around achievement hunters, trophy hunters, and completionists.  
- Serve as the main hub for announcements, roadmap updates, and changelogs.  
- Collect feedback on features, UX, bugs, and future ideas.  
- Provide a place where players can share achievements and celebrate mastery together.  
- Enable AI-powered achievement help and discussion (initially manual, later via bot).  
- Support events, challenges, and future programs (Trials, Legendary runs, etc.).  
- Connect players, creators, and the Trophi team in one central place.

This server should feel like **the home base for people who want to master their games.**

---

## 2. Core Principles

1. **Clarity first** ‚Äì Channels should be intuitive, minimal, and clearly labeled.  
2. **Scalability** ‚Äì The structure must support future expansions (more channels, events, roles) without needing a full redesign.  
3. **Identity-driven** ‚Äì Roles, emojis, and copy should reinforce the Trophi ‚Äúmastery, completion, progress‚Äù identity.  
4. **Safe, respectful space** ‚Äì Strong moderation to keep the environment welcoming, inclusive, and fun.  
5. **Growth-ready** ‚Äì The server should be ready to onboard users coming from the landing page, email, and future in-app links.  
6. **Low friction** ‚Äì Easy to navigate, easy to participate, easy to understand where to post.

## 6. Future-Proofing & Best Practices

To make it easy to expand later, follow these guidelines from the start:

- Use clear, lowercase channel names with hyphens (e.g., `#game-discussions`, not `#GameDiscussions`).  
- Use channel topics/descriptions to explain what belongs in each place.  
- Avoid creating too many channels before there is demand. Start lean.  
- Use slowmode in `#general-chat` if traffic becomes hard to moderate.  
- Keep `AI HELP` channels open but manage expectations until the bot is live (pin a message clarifying current status).  
- Keep internal/staff channels clearly separated and hidden from normal members.  
- Use roles thoughtfully: don‚Äôt create a new role for every small detail.
